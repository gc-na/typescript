# استخدام الأمر "break" في TypeScript: الدليل الشامل

## ملخص
الأمر "break" في TypeScript هو تعليمة تستخدم لإنهاء تنفيذ حلقة أو جملة switch، مما يسمح بالتحكم في تدفق البرنامج بطريقة فعالة.

## الوثائق
تُستخدم تعليمة "break" في TypeScript لإنهاء تنفيذ حلقة (مثل for، while، do...while) أو للخروج من جملة switch. بدون استخدام "break"، ستستمر الحلقة في التنفيذ حتى تنتهي شرطها، وهذا قد يؤدي إلى حلقات لا نهائية أو سلوك غير متوقع.

### الغرض
الغرض من استخدام "break" هو التحكم في تدفق التنفيذ، مما يمنح المبرمج القدرة على إنهاء العمليات في الوقت المناسب.

### الاستخدام
يمكن استخدام "break" في السياقات التالية:
1. **داخل الحلقات**: لإنهاء الحلقة في حال تحقق شرط معين.
2. **داخل جملة switch**: للخروج من الحالة الحالية ومنع تنفيذ الحالات التالية.

## الأمثلة
### مثال 1: استخدام break في حلقة for
```typescript
for (let i = 0; i < 10; i++) {
    if (i === 5) {
        break; // إنهاء الحلقة عند الوصول إلى القيمة 5
    }
    console.log(i);
}
// الناتج: 0, 1, 2, 3, 4
```

### مثال 2: استخدام break في جملة switch
```typescript
const fruit = 'موز';

switch (fruit) {
    case 'تفاح':
        console.log('تفاح');
        break;
    case 'موز':
        console.log('موز');
        break;
    default:
        console.log('فاكهة غير معروفة');
}
// الناتج: موز
```

## الشرح
### الأخطاء الشائعة
1. **عدم استخدام break**: عند عدم استخدام "break" في جملة switch، سيستمر تنفيذ جميع الحالات التالية، مما قد يؤدي إلى نتائج غير متوقعة.
2. **حلقات لا نهائية**: في الحلقات، قد يؤدي عدم استخدام "break" أو عدم وجود شرط صحيح للخروج من الحلقة إلى حدوث حلقة لا نهائية.

### ملاحظات إضافية
- يمكن استخدام "break" مع الكلمات الرئيسية "label" لتحديد أي حلقة تريد إنهاءها في حالة وجود حلقات متداخلة.
- يُفضل دائمًا استخدام "break" في الأماكن الصحيحة للحفاظ على وضوح الكود وسهولة صيانته.

## ملخص جملة واحدة
الأمر "break" في TypeScript يستخدم لإنهاء تنفيذ الحلقات وجمل switch، مما يوفر تحكمًا فعالًا في تدفق البرنامج.
