# استخدام الكلمة المفتاحية "new" في TypeScript: فهم شامل

## ملخص
تعتبر الكلمة المفتاحية "new" في TypeScript أداة أساسية تُستخدم لإنشاء كائنات جديدة من الفئات (classes). يتيح لك استخدام "new" إنشاء مثيلات جديدة مع الحفاظ على أنواع البيانات القوية التي يتيحها TypeScript.

## الوثائق
تستخدم الكلمة المفتاحية "new" في TypeScript لإنشاء مثيلات جديدة من الفئات. عند استخدام "new"، يتم استدعاء الدالة البانية (constructor) للفئة المحددة، مما يسمح لك بتهيئة الكائن الجديد بالخصائص المطلوبة.

### الغرض
- إنشاء كائنات جديدة من الفئات.
- تفعيل الدالة البانية للفئة، مما يسمح بتهيئة الكائن بالخصائص الأولية.

### الاستخدام
تُستخدم الكلمة المفتاحية "new" بالطريقة التالية:
```typescript
let instance = new ClassName(arguments);
```
حيث `ClassName` هي اسم الفئة و`arguments` هي القيم التي يتم تمريرها إلى الدالة البانية.

### التفاصيل
- يجب أن تكون الفئة مُعرفة مسبقًا.
- يمكن استخدام "new" مع الفئات التي تحتوي على دوال بنائية متقدمة.
- يمكن أن تحتوي الفئات على خصائص ودوال يمكن الوصول إليها من خلال المثيل.

## الأمثلة
### مثال أساسي
```typescript
class Person {
    name: string;
    age: number;

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }
}

let john = new Person("John Doe", 30);
console.log(john.name); // John Doe
console.log(john.age);  // 30
```

### مثال مع دوال إضافية
```typescript
class Animal {
    species: string;

    constructor(species: string) {
        this.species = species;
    }

    speak() {
        console.log(`${this.species} makes a noise.`);
    }
}

let dog = new Animal("Dog");
dog.speak(); // Dog makes a noise.
```

## الشرح
عند استخدام الكلمة المفتاحية "new"، يجب الانتباه إلى بعض النقاط:
- تأكد من أن الفئة تحتوي على دالة بنائية (constructor) مناسبة لتجنب الأخطاء.
- إذا حاولت إنشاء مثيل لفئة بدون دالة بنائية، ستواجه خطأ في وقت التشغيل.
- يجب أن تكون الأنواع المتوقعة في الدالة البنائية متوافقة مع القيم التي تمررها.

## ملخص بجملة واحدة
تستخدم الكلمة المفتاحية "new" في TypeScript لإنشاء مثيلات جديدة من الفئات، مما يسهل العمل مع الكائنات بطريقة منظمة وقوية.
