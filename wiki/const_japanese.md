# TypeScriptにおけるconstの使い方と特徴

## 概要
TypeScriptにおける`const`は、定数を宣言するためのキーワードです。これは、変数に再代入を許可せず、そのスコープ内での値の不変性を保証します。`const`は、コードの可読性と保守性を高めるための重要な要素です。

## ドキュメント
### 目的
`const`を使用することで、変数に対する誤った再代入を回避し、意図しないエラーを防ぐことができます。特に、オブジェクトや配列を扱う際に、参照の不変性を保証するために役立ちます。

### 使用法
`const`キーワードは、変数を宣言する際に使用します。次のように書きます。

```typescript
const variableName = value;
```

この宣言により、`variableName`はスコープ内で不変の値を持つことになります。値を再代入しようとすると、コンパイルエラーが発生します。

### 詳細
- `const`で宣言された変数は、ブロックスコープを持ちます。つまり、変数はその宣言されたブロック内でのみ有効です。
- 参照型（オブジェクトや配列）に対して`const`を使用した場合、オブジェクトや配列の内容を変更することは可能ですが、変数自体を別のオブジェクトや配列に再代入することはできません。

## 例
以下は、`const`の基本的な使用例です。

### 基本的な使用例

```typescript
const pi = 3.14;
console.log(pi); // 3.14

// 再代入するとエラーが発生します
// pi = 3.14159; // エラー: Cannot assign to 'pi' because it is a constant.

const numbers = [1, 2, 3];
numbers.push(4); // OK: 配列の内容は変更可能
console.log(numbers); // [1, 2, 3, 4]

// 再代入はできません
// numbers = [5, 6, 7]; // エラー: Cannot assign to 'numbers' because it is a constant.
```

## 説明
`const`を使用する際の一般的な落とし穴や注意点は以下の通りです。

- **再代入の禁止**: `const`は再代入を禁止しますが、オブジェクトや配列のプロパティや要素は変更可能です。これは、`const`が「不変」を意味するものではなく、変数の参照が不変であることを意味します。
- **スコープについて**: `const`はブロックスコープを持つため、例えばループ内で宣言した`const`変数はループ外でアクセスできません。
- **初期化の必要性**: `const`で宣言する際には、必ず変数を初期化する必要があります。初期化しないとエラーになります。

## 一文要約
TypeScriptの`const`は、変数の再代入を防ぎ、ブロックスコープで不変性を確保するためのキーワードです。
