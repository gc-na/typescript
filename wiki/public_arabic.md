# الكلمة المفتاحية "public" في TypeScript: فهم الاستخدام والأهمية

## ملخص
تعتبر الكلمة المفتاحية "public" في TypeScript من العناصر الأساسية التي تحدد مستوى الوصول إلى الخصائص والطرق داخل الكائنات. تستخدم لتحديد أن العنصر يمكن الوصول إليه من أي مكان في البرنامج.

## الوثائق
### الغرض
تستخدم الكلمة "public" لتحديد عناصر الكائنات (properties وmethods) التي يمكن الوصول إليها من خارج الكائن. هذا يعني أنه يمكن لأي كود في البرنامج التعامل مع هذه العناصر بحرية.

### الاستخدام
عند تعريف كائن أو فئة (class) في TypeScript، يمكن استخدام "public" لتحديد الخصائص والطرق. بشكل افتراضي، تكون جميع الخصائص والطرق في TypeScript عامة (public) ما لم يتم تحديد مستوى وصول آخر.

### التفاصيل
- **تعريف**: يمكن استخدام "public" مع الخصائص والطرق في الفئات.
- **المستوى الافتراضي**: إذا لم يتم تحديد مستوى الوصول، يعتبر "public" هو المستوى الافتراضي.
- **الوصول**: يمكن الوصول إلى الخصائص والطرق العامة من أي مكان، مما يجعلها مناسبة للاستخدام العام.

## أمثلة
### مثال 1: تعريف خاصية عامة
```typescript
class Person {
    public name: string;

    constructor(name: string) {
        this.name = name;
    }
}

const person = new Person("Ali");
console.log(person.name); // يطبع "Ali"
```

### مثال 2: تعريف طريقة عامة
```typescript
class Calculator {
    public add(x: number, y: number): number {
        return x + y;
    }
}

const calculator = new Calculator();
console.log(calculator.add(5, 3)); // يطبع "8"
```

## الشرح
### الأخطاء الشائعة
- **عدم استخدام "public"**: إذا تم تعريف خاصية أو طريقة بدون تحديد مستوى الوصول، فإنها ستكون عامة بشكل افتراضي، ولكن من الأفضل توضيح ذلك.
- **تجاوز مستوى الوصول**: يجب الانتباه إلى أن استخدام "public" يعني أن أي كود يمكنه الوصول إلى الخاصية أو الطريقة، مما قد يؤدي إلى تغييرات غير متوقعة في البيانات.

### ملاحظات إضافية
- يعتبر استخدام "public" جزءًا من مفهوم البرمجة الكائنية (OOP) ويساعد في تنظيم الكود وزيادة قابليته للصيانة.
- في بعض الأحيان، قد يكون من الأفضل استخدام "private" أو "protected" لحماية البيانات الحساسة من التلاعب الخارجي.

## ملخص من سطر واحد
تستخدم الكلمة المفتاحية "public" في TypeScript لتحديد الخصائص والطرق التي يمكن الوصول إليها بحرية من أي مكان في البرنامج.
