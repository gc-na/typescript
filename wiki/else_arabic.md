# استخدام جملة "else" في TypeScript: دليل شامل

## ملخص
جملة "else" في TypeScript تُستخدم لتوفير مسار بديل للتنفيذ عندما لا يتحقق الشرط المحدد في جملة "if". تُعد هذه الجملة جزءًا أساسيًا من التحكم في تدفق البرنامج.

## الوثائق
تُستخدم جملة "else" في TypeScript لتحديد ما يجب القيام به إذا كان الشرط في جملة "if" غير صحيح. توفر "else" طريقة لإنشاء منطق بديل، مما يساعد على تحسين تدفق البرنامج وجعله أكثر تفاعلية.

### الغرض
الغرض من جملة "else" هو السماح للمطورين بتنفيذ كود مختلف بناءً على نتيجة شرط معين. هذا يسهل على المطورين كتابة برامج أكثر تعقيدًا دون الحاجة إلى استخدام هياكل تحكم إضافية.

### الاستخدام
يمكن استخدام جملة "else" بعد جملة "if" مباشرة. إذا كان الشرط في "if" غير صحيح، يتم تنفيذ الكود داخل كتلة "else".

### التركيب
```typescript
if (condition) {
    // الكود الذي يتم تنفيذه إذا كان الشرط صحيحًا
} else {
    // الكود الذي يتم تنفيذه إذا كان الشرط غير صحيح
}
```

## الأمثلة
### مثال 1: استخدام بسيط لجملة "else"
```typescript
let number: number = 10;

if (number > 5) {
    console.log("العدد أكبر من 5");
} else {
    console.log("العدد 5 أو أقل");
}
```

### مثال 2: استخدام "if-else" متعدد
```typescript
let grade: number = 75;

if (grade >= 90) {
    console.log("ممتاز");
} else if (grade >= 75) {
    console.log("جيد جداً");
} else {
    console.log("تحتاج إلى تحسين");
}
```

## الشرح
### الأخطاء الشائعة
- **نسيان الأقواس**: يجب التأكد من استخدام الأقواس بشكل صحيح حول الشروط وكتل التعليمات البرمجية.
- **الاستخدام غير المناسب**: يمكن استخدام "else" فقط بعد جملة "if". عدم وجود جملة "if" قبل "else" سيؤدي إلى حدوث خطأ في وقت التحويل.

### ملاحظات إضافية
- يمكن دمج "else" مع "if" في بنى أكثر تعقيدًا مثل "else if" مما يسمح بتقييم شروط متعددة.
- يستخدم "else" غالبًا في التحكم في تدفق البرنامج، مما يجعله أداة قوية في تطوير التطبيقات.

## ملخص جملة واحدة
تُستخدم جملة "else" في TypeScript لتوفير مسار بديل للتنفيذ عندما لا يتحقق الشرط في جملة "if"، مما يسهل التحكم في تدفق البرنامج.
