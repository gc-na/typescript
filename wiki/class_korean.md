# TypeScript 클래스: 객체 지향 프로그래밍의 기초

## 개요
TypeScript의 클래스는 객체 지향 프로그래밍 패러다임을 지원하는 중요한 기능으로, 코드의 재사용성과 구조화된 설계를 가능하게 합니다. 클래스는 객체를 정의하고 생성하는 데 사용되며, 속성(property)과 메서드(method)를 포함할 수 있습니다.

## 문서화
TypeScript의 클래스는 ES6(ECMAScript 2015)에서 도입된 문법을 기반으로 하며, 객체 지향 언어의 여러 개념(상속, 캡슐화 등)을 지원합니다. 클래스는 다음과 같은 주요 요소를 포함합니다:

- **속성(Property)**: 클래스의 상태를 나타내는 변수입니다.
- **메서드(Method)**: 클래스의 동작을 정의하는 함수입니다.
- **생성자(Constructor)**: 클래스의 인스턴스를 생성할 때 호출되는 특수한 메서드입니다.
- **상속(Inheritance)**: 기존 클래스로부터 새로운 클래스를 생성하여 코드의 재사용성을 높입니다.

### 사용법
TypeScript에서 클래스를 정의하기 위한 기본 문법은 다음과 같습니다:

```typescript
class 클래스이름 {
    // 속성 정의
    속성명: 타입;

    // 생성자
    constructor(매개변수: 타입) {
        this.속성명 = 매개변수;
    }

    // 메서드 정의
    메서드명(매개변수: 타입): 반환타입 {
        // 메서드 내용
    }
}
```

## 예제
다음은 TypeScript에서 클래스를 사용하는 간단한 예제입니다:

```typescript
class 자동차 {
    // 속성
    브랜드: string;
    모델: string;

    // 생성자
    constructor(브랜드: string, 모델: string) {
        this.브랜드 = 브랜드;
        this.모델 = 모델;
    }

    // 메서드
    정보(): string {
        return `브랜드: ${this.브랜드}, 모델: ${this.모델}`;
    }
}

// 클래스 인스턴스 생성
const 내차 = new 자동차("현대", "아반떼");
console.log(내차.정보()); // 출력: 브랜드: 현대, 모델: 아반떼
```

## 설명
TypeScript 클래스를 사용할 때 유의해야 할 몇 가지 일반적인 함정과 주의사항은 다음과 같습니다:

1. **접근 제한자**: 클래스의 속성과 메서드는 `public`, `private`, `protected` 접근 제한자를 가질 수 있습니다. `private`로 설정하면 해당 클래스 외부에서 접근이 불가능합니다.

2. **상속의 오용**: 클래스 상속을 사용할 때, 불필요하게 복잡한 구조를 만들지 않도록 주의해야 합니다. 상속을 잘못 사용할 경우 코드가 복잡해지고 유지보수하기 어려워질 수 있습니다.

3. **정적 메서드와 속성**: 클래스의 메서드와 속성에 `static` 키워드를 사용하면 인스턴스가 아닌 클래스 자체에 속하게 됩니다. 이를 활용하면 유틸리티 함수나 공통된 데이터를 관리할 수 있습니다.

## 한 문장 요약
TypeScript의 클래스는 객체 지향 프로그래밍을 지원하며, 속성과 메서드를 포함하여 코드의 재사용성을 높이는 강력한 구조체입니다.
