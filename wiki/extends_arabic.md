# استخدام الكلمة المفتاحية "extends" في TypeScript

## الملخص
تُستخدم الكلمة المفتاحية "extends" في TypeScript لتحديد العلاقات بين الأنواع، سواءً في الوراثة أو في قيود الأنواع. تساعد هذه الكلمة في توسيع الفئات أو الأنواع، مما يسهل إعادة استخدام الكود وتحسين تنظيمه.

## الوثائق
### الغرض
تُستخدم "extends" لتحديد أن نوعًا أو فئة ما يُشتق من نوع آخر، مما يسمح بإنشاء هياكل بيانات أكثر تعقيدًا وقابلية لإعادة الاستخدام.

### الاستخدام
1. **الوراثة في الفئات**: يمكن استخدام "extends" لإنشاء فئة جديدة تعتمد على فئة أخرى، مما يوفر إمكانية الوصول إلى خصائص وطرق الفئة الأم.
2. **قيود الأنواع**: يمكن استخدامها لتحديد أن نوعًا معينًا يجب أن يكون نوعًا فرعيًا من نوع آخر.

### التفاصيل
- عند استخدام "extends" مع الفئات، يتم إنشاء علاقة وراثية بين الفئة الابنة والفئة الأم.
- عند استخدامها مع الأنواع، فإنها تحدد أنه يجب أن يحتوي النوع الجديد على جميع خصائص النوع الأساسي.

## أمثلة
### مثال 1: الوراثة في الفئات
```typescript
class Animal {
    name: string;
    constructor(name: string) {
        this.name = name;
    }
    speak() {
        console.log(`${this.name} makes a noise.`);
    }
}

class Dog extends Animal {
    speak() {
        console.log(`${this.name} barks.`);
    }
}

let dog = new Dog("Rex");
dog.speak(); // Rex barks.
```

### مثال 2: قيود الأنواع
```typescript
interface Person {
    name: string;
}

function greet<T extends Person>(person: T): void {
    console.log(`Hello, ${person.name}`);
}

greet({ name: "Ali" }); // Hello, Ali
```

## الشرح
- **المشاكل الشائعة**: يمكن أن تحدث أخطاء في الوراثة إذا تم استخدام "extends" بشكل غير صحيح، مثل عدم تضمين الخصائص المطلوبة في الفئة الابنة.
- **نقاط يجب الانتباه لها**: تأكد من أن الأنواع المستخدمة تتوافق مع القيود المحددة. قد يؤدي استخدام الأنواع غير المتوافقة إلى أخطاء في وقت التشغيل.

## ملخص بجملة واحدة
تعمل الكلمة المفتاحية "extends" في TypeScript على إنشاء علاقات وراثية بين الفئات وتحديد القيود على الأنواع، مما يعزز إمكانية إعادة استخدام الكود.
