# جملة switch في TypeScript: استخداماتها وأمثلة توضيحية

## ملخص
جملة `switch` في TypeScript هي عبارة عن بنية تحكم تساعد على اتخاذ القرارات بناءً على قيمة متغيرة، مما يسهل قراءة الكود وصيانته عند التعامل مع عدة حالات بدلاً من الجمل الشرطية المتعددة.

## الوثائق
تُستخدم جملة `switch` لمقارنة قيمة متغيرة مع مجموعة من القيم المحتملة، حيث يمكن تنفيذ كود معين بناءً على تطابق تلك القيم. تكمن فائدة استخدام `switch` في تنظيم الشيفرة وتقليل التعقيد مقارنة باستخدام جملة `if...else if`.

### التركيب
```typescript
switch (expression) {
    case value1:
        // تنفيذ الكود إذا كانت القيمة تطابق value1
        break;
    case value2:
        // تنفيذ الكود إذا كانت القيمة تطابق value2
        break;
    // يمكنك إضافة المزيد من الحالات
    default:
        // تنفيذ الكود إذا لم تتطابق أي من القيم
        break;
}
```

### الغرض
- تسهيل إدارة الشيفرة عند وجود العديد من الاحتمالات.
- تحسين قابلية القراءة والصيانة للكود.

### الاستخدام
تستخدم جملة `switch` في الحالات التي:
- تحتاج إلى اتخاذ قرار بناءً على قيمة معينة.
- ترغب في مقارنة قيمة واحدة مع عدة خيارات مختلفة.

## أمثلة
### مثال أساسي
```typescript
let fruit: string = "تفاح";

switch (fruit) {
    case "موز":
        console.log("هذا موز");
        break;
    case "تفاح":
        console.log("هذا تفاح");
        break;
    case "برتقال":
        console.log("هذا برتقال");
        break;
    default:
        console.log("فاكهة غير معروفة");
}
```

### مثال مع القيم العددية
```typescript
let rating: number = 3;

switch (rating) {
    case 1:
        console.log("تقييم سيء");
        break;
    case 2:
        console.log("تقييم متوسط");
        break;
    case 3:
        console.log("تقييم جيد");
        break;
    case 4:
        console.log("تقييم ممتاز");
        break;
    default:
        console.log("تقييم غير معروف");
}
```

## الشرح
### الأخطاء الشائعة
- **نسيان جملة `break`:** إذا لم تقم بإضافة جملة `break` بعد كل حالة، سيتابع تنفيذ الكود في الحالات التالية حتى يصل إلى `break` أو نهاية الجملة. هذا يمكن أن يؤدي إلى نتائج غير متوقعة.
- **عدم استخدام `default`:** عدم وجود حالة `default` يجعل من الصعب معالجة القيم غير المتوقعة.
- **استخدام قيم غير متوافقة:** تأكد من أن القيم المقارنة متوافقة من حيث النوع. إذا كانت `expression` من نوع `string`, يجب أن تكون القيم في `case` أيضًا من نوع `string`.

## ملخص من سطر واحد
جملة `switch` في TypeScript هي وسيلة فعالة لتنظيم الشيفرة والتعامل مع عدة حالات بناءً على قيمة متغيرة.
