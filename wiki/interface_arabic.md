# الواجهة (Interface) في TypeScript: دليل شامل

## ملخص
تُستخدم الواجهة (Interface) في TypeScript لتحديد شكل الكائنات، مما يساعد في تحقيق التحقق من الأنواع وتسهيل التفاعل بين الأجزاء المختلفة في التطبيق.

## الوثائق
### الغرض
تُعتبر الواجهة (Interface) في TypeScript وسيلة لتعريف بنية الكائنات. تتيح لك تحديد أنواع الخصائص والطرق التي يجب أن تحتوي عليها الكائنات، مما يسهل عملية التحقق من الصحة أثناء الكتابة.

### الاستخدام
تُستخدم الواجهات بشكل شائع لتعريف شكل العناصر التي سيتم تمريرها إلى الدوال أو التي ستحصل عليها من الدوال. يمكن أن تتضمن الواجهات خصائص وطرق، ويمكن أيضًا أن ترث من واجهات أخرى.

### التفاصيل
- **تعريف Interface**: تُستخدم الكلمة المفتاحية `interface` لتحديد واجهة جديدة.
- **خصائص**: يمكن أن تحتوي الواجهة على خصائص مع تحديد نوعها.
- **طرق**: يمكن أيضًا تحديد طرق داخل الواجهة، دون الحاجة إلى تقديم تنفيذ.
- **التوريث**: يمكن للواجهات أن ترث من واجهات أخرى، مما يسهل إعادة استخدام التعليمات البرمجية.

## الأمثلة
### مثال أساسي لتعريف واجهة
```typescript
interface شخص {
    الاسم: string;
    العمر: number;
}

const شخص1: شخص = {
    الاسم: "أحمد",
    العمر: 25
};
```

### مثال على واجهة مع طرق
```typescript
interface حيوانات {
    الصوت(): string;
}

class كلب implements حيوانات {
    الصوت(): string {
        return "نباح";
    }
}

const كلبي = new كلب();
console.log(كلبي.صوت()); // نباح
```

## الشرح
### الأخطاء الشائعة
- **نسيان النوع**: قد يحدث خطأ في حالة عدم تحديد نوع الخصائص بشكل صحيح، مما يؤدي إلى أخطاء في الوقت الذي يتم فيه استخدام الواجهة.
- **عدم تنفيذ الواجهة**: إذا تم تعريف واجهة ولكن لم يتم تنفيذها في كائن أو فئة، فإن ذلك سيؤدي إلى عدم الاستفادة الكاملة من الفوائد التي تقدمها TypeScript.
- **تعارض الأسماء**: يمكن أن يحدث تعارض في الأسماء إذا تم تعريف واجهات بنفس الاسم في نطاقات مختلفة، مما قد يسبب لبسًا.

## ملخص من جملة واحدة
الواجهة (Interface) في TypeScript تُستخدم لتحديد شكل الكائنات، مما يسهل التحقق من الأنواع والتنظيم في البرمجة.
