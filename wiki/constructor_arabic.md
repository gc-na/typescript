# المُنشئ (Constructor) في TypeScript: فهم أساسياته واستخداماته

## ملخص
المُنشئ (Constructor) في TypeScript هو دالة خاصة تُستخدم لإنشاء كائنات من فئة معينة. يُعتبر المُنشئ نقطة الانطلاق لإنشاء كائنات جديدة، حيث يُمكنك تخصيص الخصائص الأولية للكائن عند إنشائه.

## الوثائق
### الغرض
يُستخدم المُنشئ في TypeScript لتحديد كيفية إنشاء كائنات جديدة من فئة معينة. يسمح لك بإعداد الخصائص الأولية للكائن وتحديد السلوكيات الأساسية المتعلقة به.

### الاستخدام
يتم تعريف المُنشئ داخل الفئة باستخدام الكلمة المفتاحية `constructor`. يُمكن أن يحتوي المُنشئ على معاملات تُستخدم لتعيين القيم الأولية لخصائص الكائن. 

### التفاصيل
- يُمكن أن يحتوي المُنشئ على أي عدد من المعاملات.
- يتم استدعاء المُنشئ تلقائيًا عند إنشاء كائن جديد.
- يُمكن أن يحتوي المُنشئ على منطق إضافي لتنفيذ إجراءات معينة عند إنشاء الكائن.

## أمثلة
### مثال بسيط
```typescript
class شخص {
    الاسم: string;
    العمر: number;

    constructor(الاسم: string, العمر: number) {
        this.الاسم = الاسم;
        this.العمر = العمر;
    }

    تقديم() {
        console.log(`مرحبًا، أنا ${this.الاسم} وأبلغ من العمر ${this.العمر} عامًا.`);
    }
}

const شخص1 = new شخص("أحمد", 30);
شخص1.تقديم(); // مرحبًا، أنا أحمد وأبلغ من العمر 30 عامًا.
```

### مثال مع خيارات افتراضية
```typescript
class سيارة {
    العلامة: string;
    الموديل: string;
    السنة: number;

    constructor(العلامة: string, الموديل: string, السنة: number = 2020) {
        this.العلامة = العلامة;
        this.mوديل = الموديل;
        this.السنة = السنة;
    }
}

const سيارتي = new سيارة("تويوتا", "كامري");
console.log(سيارتي); // سيارة { العلامة: 'تويوتا', الموديل: 'كامري', السنة: 2020 }
```

## الشرح
### الأخطاء الشائعة
- **عدم تضمين الكلمة المفتاحية `constructor`**: يجب أن تُستخدم هذه الكلمة لتحديد المُنشئ بشكل صحيح.
- **عدم استخدام `this`**: يجب استخدام `this` للإشارة إلى خصائص الكائن الحالي داخل المُنشئ.
- **تجاوز عدد المعاملات**: يجب أن يتطابق عدد المعاملات المرسلة عند إنشاء الكائن مع عدد المعاملات المعرّفة في المُنشئ.

### ملاحظات إضافية
- يُمكن أن تحتوي الفئات على أكثر من مُنشئ باستخدام **المُنشئ المفرط** (Overloading) لتوفير خيارات مختلفة لتهيئة الكائنات.
- يُعتبر المُنشئ نقطة انطلاق مهمة لتطبيق أنماط التصميم المختلفة، مثل نمط **المصنع** (Factory Pattern).

## ملخص في سطر واحد
المُنشئ في TypeScript هو دالة خاصة تُستخدم لتهيئة كائنات الفئة وتحديد خصائصها الأولية.
