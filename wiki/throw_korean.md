# TypeScript에서의 throw: 오류 처리의 핵심

## 개요
TypeScript에서 `throw`는 오류를 발생시키는 데 사용되는 명령어입니다. 이는 프로그램의 실행 중에 예외 상황을 처리하고, 개발자가 정의한 조건에 따라 사용자 정의 오류를 발생시키는 데 유용합니다.

## 문서
`throw` 명령어는 JavaScript와 TypeScript에서 동일하게 작동하며, 특정 조건이 충족되지 않을 때 오류를 발생시켜 프로그램의 흐름을 제어합니다. 예를 들어, 잘못된 입력값이나 비즈니스 로직에 위배되는 경우에 사용됩니다.

### 목적
- 오류의 발생을 명시적으로 나타내어, 프로그램의 예외 처리를 쉽게 할 수 있도록 합니다.
- 개발자가 정의한 조건에 따른 오류 메시지를 제공하여 디버깅을 용이하게 합니다.

### 사용법
`throw`는 다음과 같은 형식으로 사용됩니다:

```typescript
throw new Error("오류 메시지");
```

이렇게 사용하면, 새로운 `Error` 객체가 생성되고, 해당 오류 메시지가 포함됩니다. 이 오류는 호출 스택을 따라 전파되어, 적절한 예외 처리기에서 잡히거나 프로그램이 종료되는 원인이 될 수 있습니다.

## 예제
기본적인 `throw` 사용 예제는 다음과 같습니다:

```typescript
function checkPositiveNumber(num: number): void {
    if (num < 0) {
        throw new Error("음수는 허용되지 않습니다.");
    }
    console.log("양수입니다:", num);
}

try {
    checkPositiveNumber(-5);
} catch (error) {
    console.error("오류 발생:", error.message);
}
```

위의 예제에서, `checkPositiveNumber` 함수는 양수가 아닌 경우 오류를 발생시킵니다. 이 오류는 `try-catch` 블록에서 처리됩니다.

## 설명
`throw` 사용 시 주의해야 할 몇 가지 사항은 다음과 같습니다:

- **명확한 오류 메시지**: 오류 메시지는 문제를 이해하는 데 중요한 역할을 하므로, 명확하고 구체적인 메시지를 제공하는 것이 좋습니다.
- **예외 처리**: `throw`로 발생한 오류는 반드시 `try-catch` 블록으로 처리해야 합니다. 그렇지 않으면 프로그램이 비정상 종료될 수 있습니다.
- **타입스크립트의 타입 시스템**: TypeScript에서는 `throw`로 발생시키는 오류의 타입을 정의할 수 있습니다. 예를 들어, 사용자 정의 오류 클래스를 만들어 사용할 수도 있습니다.

## 한 줄 요약
TypeScript의 `throw` 명령어는 특정 조건에서 오류를 발생시켜 프로그램의 흐름을 제어하고, 명확한 예외 처리를 가능하게 합니다.
