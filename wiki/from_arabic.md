# استخدام الكلمة "from" في TypeScript: دليل شامل

## ملخص
تُستخدم الكلمة "from" في TypeScript للإشارة إلى مصدر استيراد الوحدات أو الملفات. يعد هذا جزءًا أساسيًا من نظام الوحدات في JavaScript وTypeScript، مما يسهل تنظيم الكود وإعادة استخدامه.

## الوثائق
تُستخدم الكلمة "from" في عبارة الاستيراد لجلب المكونات، الدوال، أو الكائنات من ملفات أو مكتبات أخرى. يتم ذلك عادةً باستخدام تركيبة `import { component } from 'module'`.

### الغرض
يهدف استخدام "from" إلى تسهيل التعامل مع الأكواد من ملفات متعددة، مما يسمح للمطورين بتنظيم مشاريعهم بشكل أفضل.

### الاستخدام
تُستخدم الكلمة "from" في عبارات الاستيراد كما يلي:

```typescript
import { component } from 'module';
```

يتطلب هذا بناءً على نظام الوحدات في TypeScript، حيث يتم استيراد `component` من الملف أو المكتبة المشار إليها بـ 'module'.

## الأمثلة
### مثال 1: استيراد دالة
```typescript
import { calculateArea } from './geometry';

const area = calculateArea(5);
console.log(area);
```

### مثال 2: استيراد كائن كامل
```typescript
import * as geometry from './geometry';

const area = geometry.calculateArea(5);
console.log(area);
```

### مثال 3: استيراد افتراضي
```typescript
import DefaultComponent from './DefaultModule';
```

## الشرح
### الأخطاء الشائعة
- **استخدام المسارات غير الصحيحة**: تأكد من أن المسار المحدد في عبارة الاستيراد صحيح. يمكن أن يؤدي الخطأ في المسار إلى عدم العثور على الوحدة.
- **استيراد غير موجود**: تأكد من أن المكونات أو الدوال المراد استيرادها موجودة بالفعل في الملف المستهدف، وإلا ستظهر أخطاء في وقت التشغيل.
- **تعارض الأسماء**: إذا كان لديك مكونين بنفس الاسم في ملفات مختلفة، فقد تحتاج إلى استخدام أسماء مستعارة لتجنب التعارضات.

### ملاحظات إضافية
- يمكن استخدام "from" في الاستيراد الجزئي أو الكامل، حسب الحاجة. 
- تأكد من أن الوحدات التي تستوردها تتوافق مع نوع البيانات التي تعمل معها في مشروعك.

## ملخص بجملة واحدة
تساعد الكلمة "from" في TypeScript على استيراد المكونات من ملفات أو مكتبات خارجية، مما يسهل تنظيم وإعادة استخدام الكود.
