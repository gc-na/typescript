# TypeScriptにおける「super」キーワードの使い方

## 概要
TypeScriptにおける「super」キーワードは、親クラスのメソッドやプロパティにアクセスするために使用されます。この機能は、クラスの継承を利用する際に非常に便利です。

## ドキュメンテーション
### 目的
「super」は、子クラスから親クラスのメソッドやプロパティを呼び出すための特別なキーワードです。TypeScriptでは、クラスの継承を利用することで、コードの再利用性を高めることができます。

### 使用法
「super」は、コンストラクタ内やメソッド内で使用されます。親クラスのコンストラクタを呼び出す際には、`super()`を使い、メソッドを呼び出す際には、`super.methodName()`の形式で使用します。

### 詳細
- **コンストラクタの呼び出し**: 子クラスのコンストラクタ内で必ず`super()`を呼び出す必要があります。
- **メソッドのオーバーライド**: 子クラスで親クラスのメソッドをオーバーライドする際に、親クラスのメソッドを`super`で呼び出すことができます。

## 例
```typescript
class Animal {
    constructor(public name: string) {}

    speak() {
        console.log(`${this.name}が鳴いています。`);
    }
}

class Dog extends Animal {
    constructor(name: string) {
        super(name); // 親クラスのコンストラクタを呼び出す
    }

    speak() {
        super.speak(); // 親クラスのメソッドを呼び出す
        console.log(`${this.name}はワンワンと鳴きました。`);
    }
}

const dog = new Dog('ポチ');
dog.speak();
```

## 説明
- **一般的な落とし穴**: `super()`をコンストラクタの最初の行以外で呼び出すと、エラーが発生します。これにより、親クラスの初期化が行われないためです。
- **メソッドの呼び出し**: オーバーライドしたメソッド内で`super`を呼び出さないと、親クラスの実装が実行されないため、意図した動作にならない場合があります。
