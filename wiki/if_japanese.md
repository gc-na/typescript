# TypeScriptにおける「if」文の使い方

## 概要
TypeScriptにおける「if」文は、条件に基づいてプログラムの実行フローを制御するための基本的な構文です。この文を使用することで、特定の条件が真である場合にのみ特定のコードブロックを実行することができます。

## ドキュメンテーション
### 目的
「if」文は、条件式を評価し、その結果に応じて異なるコードを実行するために使用されます。TypeScriptでは、JavaScriptと同様の構文を使用して条件分岐を実装します。

### 使用法
基本的な「if」文の構文は以下の通りです。

```typescript
if (条件式) {
    // 条件が真の場合に実行されるコード
}
```

### 詳細
- **条件式**: ブール値（trueまたはfalse）を返す式です。条件が真の場合、波括弧内のコードが実行されます。
- **else文**: 「if」文には「else」部分を追加することで、条件が偽の時に実行されるコードを指定できます。

```typescript
if (条件式) {
    // 条件が真の場合
} else {
    // 条件が偽の場合
}
```

- **else if文**: 複数の条件を評価する場合は、「else if」を使うことができます。

```typescript
if (条件式1) {
    // 条件1が真の場合
} else if (条件式2) {
    // 条件2が真の場合
} else {
    // 条件1も条件2も偽の場合
}
```

## 例
### 基本的な使用例
```typescript
const number = 10;

if (number > 5) {
    console.log("5より大きいです");
} else {
    console.log("5以下です");
}
```

### 複数の条件を評価する例
```typescript
const score = 85;

if (score >= 90) {
    console.log("優秀");
} else if (score >= 75) {
    console.log("良好");
} else {
    console.log("改善が必要");
}
```

## 説明
- **型の確認**: TypeScriptは静的型付けに対応しているため、条件式内で型を確認することができます。これにより、予期しない型エラーを防ぐことができます。
- **短絡評価**: 複雑な条件式を使用する場合、短絡評価（&&や||）を利用することができますが、不適切な使用は論理エラーの原因になることがあります。
- **ネストされた「if」文**: 条件分岐が複雑になると、ネストされた「if」文が必要になることがありますが、可読性が低下する可能性があるため注意が必要です。
