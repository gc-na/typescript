# readonly في TypeScript: كيفية استخدامه وفوائده

## ملخص
تعتبر خاصية `readonly` في TypeScript أداة قوية تتيح للمطورين تحديد الخصائص غير القابلة للتغيير في الكائنات، مما يساعد في إنشاء كود أكثر أمانًا وقابلية للصيانة.

## الوثائق
تُستخدم كلمة `readonly` في TypeScript لتحديد الخصائص التي لا يمكن تعديلها بعد تعيينها. عندما يتم تعريف خاصية باستخدام `readonly`، فإنها تصبح غير قابلة للتغيير، مما يمنع الأخطاء غير المقصودة التي قد تحدث بسبب التعديلات غير المرغوب فيها.

### الغرض
الغرض من استخدام `readonly` هو تعزيز الأمان في الكود ومنع التغييرات غير المقصودة على الخصائص المهمة.

### الاستخدام
يمكن استخدام `readonly` في تعريف الكائنات، الواجهات، والمصفوفات. يُفضل استخدامه عندما ترغب في التأكد من أن القيم لن تتغير بعد تم تعيينها.

### التفاصيل
- يمكن استخدام `readonly` مع الخصائص في الكائنات والواجهات.
- يُمكن استخدام `readonly` أيضًا مع المصفوفات لحماية عناصرها من التعديل.

## الأمثلة

### مثال 1: استخدام readonly مع الخصائص
```typescript
class User {
    readonly name: string;

    constructor(name: string) {
        this.name = name;
    }
}

const user = new User("Ali");
// user.name = "Omar"; // سيتسبب هذا في خطأ لأن name خاصية readonly
```

### مثال 2: استخدام readonly مع الواجهات
```typescript
interface Point {
    readonly x: number;
    readonly y: number;
}

const point: Point = { x: 10, y: 20 };
// point.x = 15; // سيتسبب هذا في خطأ لأن x خاصية readonly
```

### مثال 3: استخدام readonly مع المصفوفات
```typescript
const numbers: readonly number[] = [1, 2, 3];
// numbers[0] = 10; // سيتسبب هذا في خطأ لأن المصفوفة readonly
```

## الشرح
من المهم أن نفهم أن استخدام `readonly` لا يعني أن الخاصية لا يمكن الوصول إليها، بل يعني أنه لا يمكن تعديلها بعد تعيينها. قد يحدث بعض الالتباس حول ما إذا كانت `readonly` تعني أن الكائن نفسه غير قابل للتغيير. في الواقع، يمكنك تغيير الخصائص القابلة للتغيير في الكائن، ولكن لا يمكنك تغيير الخاصية المحددة بـ `readonly`.

### الأخطاء الشائعة
1. **تجاوز الخصائص readonly**: محاولة تعديل خاصية `readonly` ستؤدي إلى حدوث أخطاء في وقت الترجمة، مما يؤكد على ضرورة الالتزام بقواعد TypeScript.
2. **فهم معنى readonly**: قد يعتقد البعض أن `readonly` يمنع التعديلات على الكائن بالكامل، ولكن يجب أن نتذكر أنه يمنع فقط تعديل الخصائص المحددة.

## ملخص جملة واحدة
تساعد خاصية `readonly` في TypeScript على حماية الخصائص من التعديل غير المقصود، مما يعزز أمان الكود وكفاءته.
