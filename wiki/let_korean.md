# TypeScript의 let: 블록 범위 변수를 선언하는 방법

## 개요
TypeScript에서 `let`은 블록 범위 변수를 선언하는 데 사용되는 키워드입니다. `let`은 변수를 선언할 때, 해당 변수가 정의된 블록 내에서만 유효하도록 범위를 제한하는 기능을 제공합니다. 이는 코드의 가독성을 높이고 변수 충돌을 방지하는 데 도움을 줍니다.

## 문서화
### 목적
`let` 키워드는 JavaScript ES6에서 도입된 이후 TypeScript에서도 동일하게 지원됩니다. `let`을 사용하여 변수를 선언하면, 해당 변수가 선언된 블록 내에서만 접근할 수 있으며, 이는 프로그래밍에서 범위 관련 문제를 최소화하는 데 유용합니다.

### 사용법
`let`을 사용하여 변수를 선언할 때는 다음과 같은 구문을 사용합니다:

```typescript
let variableName: type = initialValue;
```

- `variableName`: 변수의 이름
- `type`: 변수의 타입 (선택적)
- `initialValue`: 변수의 초기값 (선택적)

### 세부사항
- `let`으로 선언된 변수는 같은 블록 내에서 재선언할 수 없습니다.
- 변수의 값은 변경 가능하며, 이는 `const`와의 주요 차이점입니다.
- `let`은 함수나 루프와 같은 블록 스코프 내에서 사용될 경우, 해당 블록 내에서만 유효합니다.

## 예제
### 기본 사용 예제
```typescript
function example() {
    let x: number = 10;
    if (true) {
        let x: number = 20; // 다른 블록에서의 x
        console.log(x); // 20
    }
    console.log(x); // 10
}

example();
```

이 예제에서 두 개의 `x` 변수가 선언되지만, 각각의 블록 내에서 유효합니다.

### 반복문에서의 사용
```typescript
for (let i = 0; i < 5; i++) {
    console.log(i); // 0, 1, 2, 3, 4
}
// console.log(i); // 오류: i는 블록 범위 내에서만 유효합니다.
```

여기서 `i`는 for 루프의 블록 내에서만 접근이 가능합니다.

## 설명
`let`을 사용할 때 주의해야 할 점은 변수의 스코프입니다. `let`으로 선언한 변수는 같은 블록 내에서 재선언할 수 없고, 이는 코드의 명확성을 높입니다. 하지만 변수를 선언하기 전에 접근하면 오류가 발생하므로, 항상 변수를 먼저 선언한 후 사용하는 것이 좋습니다. 또한, `let`은 변수를 동적으로 변경할 수 있게 해주지만, 변수의 스코프와 생명 주기를 잘 이해하고 있어야 합니다.

## 한 줄 요약
