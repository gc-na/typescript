# TypeScriptの「import」文: モジュールのインポート方法と使い方

## 概要
TypeScriptにおける「import」文は、他のモジュールからコードを読み込み、再利用するための構文です。この機能により、コードの分割や管理が容易になり、プロジェクトの可読性と保守性が向上します。

## ドキュメント
TypeScriptでは、`import`文を使用して他のファイル（モジュール）から関数、クラス、変数などをインポートできます。これにより、コードの再利用が可能になり、複数のファイルに分けて開発することが推奨されます。

### 基本構文
```typescript
import { 輸入したい項目 } from './モジュール名';
```
もしくは、デフォルトエクスポートのインポート方法：
```typescript
import デフォルト名 from './モジュール名';
```

### 詳細
- インポートする対象は、エクスポートされたものだけです。エクスポートされていない項目をインポートしようとすると、コンパイルエラーが発生します。
- TypeScriptはES6のモジュールシステムをサポートしており、`import`文はその一部です。これにより、異なるファイルで定義されたコードを簡単に組み合わせることができます。
- 相対パスまたは絶対パスを使用してモジュールを指定することができますが、通常は相対パスが使われます。

## 例
以下は、TypeScriptにおける`import`文の基本的な使い方の例です。

### 例1: 名前付きエクスポートのインポート
```typescript
// module.ts
export const greeting = 'こんにちは';
export function greet(name: string) {
    return `${greeting}, ${name}!`;
}

// main.ts
import { greeting, greet } from './module';

console.log(greet('世界')); // こんにちは, 世界!
```

### 例2: デフォルトエクスポートのインポート
```typescript
// module.ts
const farewell = 'さようなら';
export default farewell;

// main.ts
import farewell from './module';

console.log(farewell); // さようなら
```

## 説明
「import」文を使う際に注意が必要な点はいくつかあります。

1. **エクスポートの有無**: インポートする項目が必ずエクスポートされていることを確認してください。エクスポートされていない項目をインポートしようとすると、コンパイルエラーが発生します。
   
2. **名前の一致**: 名前付きエクスポートをインポートする場合、インポートする名前はエクスポート時の名前と一致する必要があります。異なる名前を使う場合は、`as`キーワードを使用して別名を設定できます。

3. **循環依存**: モジュール間で循環依存が発生すると、予期しない動作を引き起こすことがあります。このため、モジュールの設計時には依存関係を意識する必要があります。

## 一文要約
TypeScriptの「import」文は、他のモジュールからコードをインポートし、プロジェクト内での再利用を可能にする強力な機能です。
