# TypeScriptの抽象クラスと抽象メソッド

## 概要
TypeScriptの「abstract」は、クラスやメソッドに対して抽象的な性質を持たせるためのキーワードです。抽象クラスは直接インスタンス化できず、サブクラスに継承されることを目的としています。この機能は、オブジェクト指向プログラミングのポリモーフィズムを実現するのに役立ちます。

## ドキュメンテーション
### 目的
抽象クラスは、共通のインターフェースを持つ複数のクラスを作成するために使用されます。これにより、異なる実装を持つクラス間での統一性を確保し、コードの再利用性を高めることができます。

### 使用法
TypeScriptで抽象クラスを定義するには、`abstract`キーワードを使用します。抽象メソッドも同様に`abstract`キーワードを使用して定義し、これを含む抽象クラスは、必ずサブクラスで実装される必要があります。

#### 構文
```typescript
abstract class ClassName {
    abstract methodName(param: Type): ReturnType;
}
```

### 詳細
- 抽象クラスは、通常のクラスと同様にプロパティやメソッドを持つことができますが、少なくとも1つの抽象メソッドを持つ必要があります。
- サブクラスは、親クラスの抽象メソッドを実装しなければなりません。これにより、サブクラスは親クラスの契約を遵守することが強制されます。

## 例
以下は、TypeScriptでの抽象クラスと抽象メソッドの基本的な使用例です。

```typescript
abstract class Animal {
    abstract makeSound(): void;

    move(): void {
        console.log("移動します");
    }
}

class Dog extends Animal {
    makeSound(): void {
        console.log("ワンワン");
    }
}

class Cat extends Animal {
    makeSound(): void {
        console.log("ニャー");
    }
}

const dog: Animal = new Dog();
dog.makeSound(); // ワンワン
dog.move();      // 移動します

const cat: Animal = new Cat();
cat.makeSound(); // ニャー
cat.move();      // 移動します
```

## 説明
- **共通のインターフェース**: 抽象クラスを使用することで、異なる動物クラスが同じメソッド（`makeSound`）を持つことが保証されます。これにより、ポリモーフィズムを利用した柔軟な設計が可能になります。
- **インスタンス化不可**: 抽象クラス自体は直接インスタンス化することができないため、抽象クラスを使用する際は必ずサブクラスを介する必要があります。

### よくある落とし穴
- 抽象メソッドを持つクラスをインスタンス化しようとすると、コンパイルエラーが発生します。
- サブクラスで抽象メソッドを実装し忘れると、サブクラスも抽象クラスとして扱われ、インスタンス化できなくなります。

## 一文要約
TypeScriptの「abstract」キーワードは、抽象クラスと抽象メソッドを定義し、オブジェクト指向プログラミングのポリモーフィズムを実現するために使用されます。
