# TypeScript의 void: 함수의 반환형 및 사용법

## 개요
TypeScript에서 `void`는 함수가 값을 반환하지 않음을 나타내는 반환 타입입니다. 주로 함수의 반환값이 필요 없는 경우에 사용됩니다.

## 문서화
`void`는 TypeScript의 기본 데이터 타입 중 하나로, 주로 다음과 같은 용도로 사용됩니다:

- **함수의 반환 타입 정의**: 함수가 값을 반환하지 않을 때 사용됩니다. 예를 들어, 이벤트 핸들러나 콜백 함수 등에서 유용합니다.
- **명시적인 타입 지정**: `void`를 사용함으로써 함수가 어떠한 값도 반환하지 않는다는 것을 명확히 할 수 있습니다.

### 사용법
`void`는 함수 선언 시 반환 타입으로 지정할 수 있습니다. 기본적인 문법은 다음과 같습니다:

```typescript
function 함수명(): void {
    // 함수 본문
}
```

이렇게 정의된 함수는 반환값이 없음을 나타냅니다.

## 예시
아래는 `void` 타입을 사용한 간단한 예시입니다.

### 예시 1: 기본 함수
```typescript
function 로그메시지(): void {
    console.log("메시지가 출력되었습니다.");
}

로그메시지(); // 콘솔에 메시지가 출력됩니다.
```

### 예시 2: 이벤트 핸들러
```typescript
function 클릭핸들러(): void {
    alert("버튼이 클릭되었습니다.");
}

document.getElementById("myButton").addEventListener("click", 클릭핸들러);
```

## 설명
`void`를 사용할 때 주의해야 할 점은 다음과 같습니다:

- **값 반환 금지**: `void`로 정의된 함수는 반환값을 가질 수 없습니다. 반환값이 있는 경우 TypeScript에서 오류가 발생합니다.
- **undefined vs null**: `void`는 함수가 반환하지 않는다는 것을 의미하며, `undefined`와는 다릅니다. `undefined`는 값으로 사용될 수 있지만, `void`는 반환값으로 사용될 수 없습니다.
- **함수 타입**: `void`는 함수의 타입으로도 사용될 수 있습니다. 예를 들어, `() => void`는 인자가 없고 반환값이 없는 함수의 타입을 의미합니다.

## 한 줄 요약
TypeScript에서 `void`는 함수가 값을 반환하지 않음을 나타내는 타입입니다.
