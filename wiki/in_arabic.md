# استخدام الكلمة المفتاحية "in" في TypeScript: دليل شامل

## ملخص
كلمة "in" في TypeScript تُستخدم للتحقق مما إذا كانت خاصية معينة موجودة في كائن ما. إنها أداة قوية للتعامل مع الكائنات وضمان وجود الخصائص قبل استخدامها.

## الوثائق
تعتبر الكلمة المفتاحية "in" واحدة من الأدوات الأساسية في TypeScript، حيث تساعد المطورين على التحقق من وجود خصائص معينة داخل الكائنات. يتم استخدامها بشكل شائع في الجمل الشرطية لتفادي الأخطاء الناتجة عن محاولة الوصول إلى خصائص غير موجودة.

### الغرض
تسهل "in" عملية التحقق من وجود خاصية في كائن، مما يساهم في كتابة كود أكثر أمانًا وقابلية للصيانة.

### الاستخدام
يمكن استخدام "in" كالتالي:
```typescript
if ('propertyName' in object) {
    // تنفيذ الكود إذا كانت الخاصية موجودة
}
```

## الأمثلة
### مثال 1: التحقق من وجود خاصية في كائن
```typescript
interface Person {
    name: string;
    age: number;
}

const person: Person = {
    name: "أحمد",
    age: 30
};

if ('age' in person) {
    console.log("الخاصية 'age' موجودة في الشخص.");
}
```

### مثال 2: استخدام "in" مع أنواع مختلفة
```typescript
type Vehicle = {
    wheels: number;
    brand?: string;
};

const car: Vehicle = {
    wheels: 4
};

if ('brand' in car) {
    console.log("العلامة التجارية موجودة في السيارة.");
} else {
    console.log("لا توجد علامة تجارية محددة.");
}
```

## الشرح
عند استخدام الكلمة المفتاحية "in"، يجب الانتباه إلى بعض النقاط المهمة:

1. **التحقق من الخصائص القابلة للتعريف**: "in" تتحقق فقط من وجود الخصائص، ولا تتحقق من القيم. لذا، إذا كانت الخاصية موجودة ولكن قيمتها `undefined`، ستعتبرها موجودة.
  
2. **الأشياء الموروثة**: "in" يتحقق أيضًا من الخصائص الموروثة من البروتوتايب، مما يعني أنه يمكن أن يعيد نتيجة إيجابية حتى لو لم يتم تعريف الخاصية مباشرة على الكائن نفسه.

3. **تجنب الأخطاء**: استخدام "in" يقلل من الأخطاء الناتجة عن الوصول إلى خصائص غير موجودة، مما يجعل الكود أكثر أمانًا.

## ملخص من جملة واحدة
تساعد الكلمة المفتاحية "in" في TypeScript المطورين على التحقق من وجود خصائص معينة في الكائنات، مما يعزز أمان الكود ويقلل من الأخطاء.
