# TypeScript 中的 const：不可變變數的使用與特性

## 摘要
在 TypeScript 中，`const` 是用於聲明不可變變數的關鍵字。這意味著一旦變數被賦值，無法再更改其值。`const` 的使用有助於提高代碼的可讀性與可維護性，並能夠避免意外的變數修改。

## 文檔
`const` 關鍵字用於聲明一個常量，這是一種在編譯時期其值不可變的變數。與 `let` 和 `var` 不同，使用 `const` 聲明的變數必須在聲明同時進行初始化。這使得代碼更具穩定性，特別是在需要保護變數不被意外修改的情況下。

### 目的
使用 `const` 可以防止變數的值被重新賦值，這在處理重要的數據時尤其有用。

### 使用方式
```typescript
const variableName: type = value;
```
- `variableName`：變數的名稱。
- `type`：變數的類型（可選）。
- `value`：變數的初始值。

### 詳細說明
1. **初始化要求**：使用 `const` 聲明的變數必須在聲明時進行初始化，否則將導致編譯錯誤。
2. **塊級作用域**：`const` 變數具有塊級作用域，這意味著它們只在其定義的塊內部可見。
3. **對象的可變性**：雖然 `const` 防止變數的重新賦值，但如果變數是對象或陣列，則對象的屬性或陣列的元素仍然可以被修改。

## 範例
```typescript
const pi: number = 3.14;
console.log(pi); // 輸出: 3.14

const numbers: number[] = [1, 2, 3];
numbers.push(4); // 可以修改陣列內容
console.log(numbers); // 輸出: [1, 2, 3, 4]

// pi = 3.14159; // 將導致編譯錯誤
```

## 解釋
- **常見陷阱**：初學者常常會混淆 `const` 和 `let`/`var`，以為 `const` 可以用於改變後的變數。需要注意的是，`const` 只防止變數的重新賦值，而非對象內部的修改。
- **使用建議**：建議在需要保護變數不被意外改寫時使用 `const`，這有助於代碼的穩定性和可讀性。

## 一句總結
在 TypeScript 中，`const` 用於聲明不可變的變數，能夠有效避免意外的值修改，增強代碼的安全性。
